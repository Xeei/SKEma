## Project

SKE-Schema is an open-source academic file-sharing platform for the SKE-22 cohort at Kasetsart University. Students share study materials, vote on posts, and earn reputation. Hosted on a Raspberry Pi via Docker + GitHub Actions.

## Commands

### Local Development
```bash
# Start DB only (required before running services locally)
docker compose up db -d

# Backend (from /backend)
npm run dev          # hot-reload dev server on :8000
npm run lint         # ESLint --max-warnings 0
npm run format:check
npm run type-check
npm test             # Jest with coverage
npm test -- --testPathPattern=auth  # run a single test file

# Frontend (from /frontend)
npm run dev          # Next.js with Turbopack on :3000
npm run lint
npm run format:check
npm run type-check
npm run build        # verify production build

# Database (from /backend)
npm run db:migrate:dev   # create + apply a new migration
npm run db:migrate       # apply existing migrations (production)
npm run db:studio        # open Prisma Studio GUI
npm run db:seed
```

### Full Stack via Docker
```bash
docker compose up --build -d   # build and start all services
docker compose ps
docker compose logs backend -f
```

## Architecture

### Monorepo Layout
```
/backend   Express API + Prisma ORM
/frontend  Next.js (App Router)
/nginx     Reverse proxy config (port 80 → frontend:3000)
/postgres  Persistent DB data volume (bind mount — never delete)
compose.yml
```

### Backend (`/backend/src`)
- **Pattern**: `routes/` → `controllers/` → `models/` (Prisma queries) — no service layer in between.
- **Auth**: Every protected route uses `authMiddleware` which verifies a JWT signed with `NEXTAUTH_SECRET`. The token is generated by NextAuth on the frontend and carries `{ sub: userId, email, name, role }`.
- **Validation**: Request bodies/queries go through `validateBody(zodSchema)` / `validateQuery(zodSchema)` middleware before hitting controllers. Schemas live in `src/schemas/index.ts`.
- **File uploads**: Multer in `src/config/upload.ts` — 50 MB limit, strict MIME allowlist. Uploaded files land in `backend/uploads/files/` which is bind-mounted in Docker so they survive redeploys.
- **Real-time**: Socket.IO (`src/socket.ts`) emits `notification` events to specific user sockets.
- **Logging**: Winston (`src/config/logger.ts`) — coloured dev output, JSON + file output in production. Morgan pipes through `morganStream`.
- **Rate limit**: 100 requests / 15 min per IP, applied to all `/api/v1/*` routes.
- **Router note**: `postreportRouter` mounts at `/api/v1/reports` — **not** `/api/v1/posts` (they were previously double-mounted and conflicted).

### Frontend (`/frontend/src`)
- **App Router** (Next.js 13+). All pages are under `src/app/`.
- **Auth flow**: Google OAuth via NextAuth → `signIn` callback validates `@ku.th` domain or `ALLOWED_EMAILS` allowlist → creates user in DB → signs a backend JWT using `NEXTAUTH_SECRET` → attaches `backendToken`, `userId`, `role` to the session.
- **API proxy**: All backend calls go through `src/app/api/proxy/[...path]/route.ts` which forwards to `BACKEND_URL` with the `backendToken` as `Authorization: Bearer`. Never call the backend directly from the browser.
- **Route protection**: `src/middleware.ts` — `/library`, `/profile`, `/community` require auth; `/admin` requires `role === ADMIN`.
- **File type validation**: `src/lib/allowedFileTypes.ts` is the single source of truth for allowed MIME types and the `accept` string used on all file inputs. Mirror any changes to `backend/src/config/upload.ts`.
- **Roles**: `STANDARD` (default), `TRUSTED`, `ADMIN`. Post creation by STANDARD users creates `PENDING` posts that require admin approval.

### Database (Prisma + PostgreSQL)
Key relationships:
- `User` → `Post` (author), `File` (uploader), `FileFolder` (owner)
- `Post` → `PostFile[]` (ordered file attachments), `PostVote[]`, `PostShare[]`, `PostReport[]`
- `FileFolder` is self-referential (parent/children) for nested folders
- `PostVote` has a unique constraint on `[postId, userId]` — voting toggles, not accumulates

### CI/CD
- **PRs**: `lint.yml` + `test.yml` run on PR to main (ubuntu-latest)
- **Push to main**: `deploy.yml` runs lint + tests first (ubuntu-latest), then `deploy` job on the self-hosted Raspberry Pi runner. Fails fast if any check fails.
- **Docker Hub**: `build-images.yml` pushes images on push to main (separate from deploy — deploy rebuilds locally on the Pi via `docker compose up --build`).

## Key Conventions

- `env.nodeEnv` and `env.corsOrigin` are lowercase in `src/config/env.ts` because the GitHub Actions secret is written as `nodeEnv`/`corsOrigin` — don't rename these to `NODE_ENV`.
- The backend JWT secret is `NEXTAUTH_SECRET` (shared with NextAuth) — not a separate `JWT_SECRET`.
- `ALLOWED_EMAILS` env var is a comma-separated list of non-KU emails permitted to sign in. When blocked, NextAuth redirects to `/auth/error?error=EmailNotAllowed`.
- Error boundaries: `src/app/error.tsx` catches page-level errors; `src/app/global-error.tsx` catches root layout errors.
